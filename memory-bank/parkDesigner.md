# Park Designer — Meebits Park 設計士ノート

> **誰向け**: Park（`src/top/**`）にオブジェクト・アトラクション・エリアを足す／動かす AI エージェントと人間。  
> **目的**: エリア追加のたびにレイアウトを壊さない。導線・境界・世界観を型で固定する。  
> **原則**: 全エリアは同じ骨格。差は「テーマ（色・地面・デコ・ゲート意匠）」だけ。

最終更新: 2026-07-24

> **配置の正**: 下記「§14 Canonical Default Layout」と `parkZones.ts` / `topConfig.ts` の現行値がデフォルト。  
> きれいな配置として凍結済み。動かすときはこの節とコードをセットで更新すること。

---

## 0. 設計思想（最重要）

Meebits Park は **海の島ではなく、テーマランドの連なり**。

ディズニーのように:

1. **各エリアは閉じた庭**（川・滝・崖・壁で外周を囲む）
2. **次のエリアへは橋を渡り、ゲートをくぐる**（境界そのものがワクワク）
3. **東西南北に常に導線を残す**（どの方位にもエリアが広がれる）
4. **全エリア同じ骨格** → 追加時はテーマ差し替えが主で、大幅な再配置をしない
5. **エリアを足すときは世界観を先に決める**（地面・色・オブジェクト・ゲート意匠を揃える）

### 廃止したもの

- 外周の **砂浜・海・椰子の島縁**（削除済み／復活させない）
- 「島を広げて家具を逃がす」場当たりレイアウト

### これから揃える境界キット（実装ロードマップ）

| 要素 | 役割 | 状態 |
|------|------|------|
| 崖 / 壁 | S/E/W の外周壁。**手前には出さない**。Plaza=石壁 / Mountain=ボクセル崖 | **両ゾーン実装済** |
| 川 | 手前以外の水路。進入不可 | **両ゾーン実装済** |
| 滝・角デコ | 手前辺に接しない角のみ | **両ゾーン実装済** |
| 橋 | カージナルのゲート／封印門前 | **本ゲート・封印門とも実装済** |
| ゲート | 橋の先。ゾーン切替。**手前に置いてよい唯一の境界物** | 双方向接続済 |
| 封印門 | 未開放カージナル。**本ゲートと同じ開口ライン＋橋＋門品質**（通過不可） | **両ゾーン実装済** |

### ハードルール: 手前クリア辺（`frontClearSides`）

**全エリア共通で北（+Z）は手前。壁・川・封印を絶対に置かない**（カメラ前を塞ぐため）。

- 禁止: 川・壁・縁石・封印門・滝・角柱・壁ランタン・その他外周デコ
- 許可: **ゲートのみ**（北にゲートがある場合だけゲート＋橋）
- 実装: 既定 `frontClearSides: ['n']`。ゾーンごとに西を手前扱いにしない
- 東西ゲートがある辺は **崖／壁を開口して門に自然につなぐ**（額縁の翼壁）
- 奥（南＝-Z）の壁・川は建物背面より外側へ

**遷移の体験順**: 園路 → 橋 → ゲート意匠 → フェード → 隣エリアのテーマ空間

Mountain 地区: テーマ `mountain`（苔・土・石・針葉樹・松明・木橋）。西門は崖壁で額縁接続。床色は `parkZoneTheme.ts`。

## 1. 世界観の軸

| 軸 | 方針 |
|----|------|
| ムード | テーマランドの「境界のワクワク」。機能ハブにしない |
| トーン | 看板はストーリーテリング |
| 外周 | S/E/W は川・滝・崖・壁。**手前（N）はゲート以外なし**（海ではない） |
| 接続 | 橋＋ゲートのみ（カージナル） |
| プレイヤー | 三人称散歩。**導線の美しさ = 遊びやすさ** |
| 拡張 | N/S/E/W のどれにも新区が付く前提 |

**やってはいけない**

- カージナル軸（東西南北の中央帯）に大型物を置く
- ゲート／橋前を家具で隠す
- **手前辺に川・壁・封印門・外周デコを置く**（ゲート以外禁止）
- エリアごとに全く違う座標系・大きさの「特例レイアウト」を増やす
- 海・砂浜を外周に戻す

## 2. 共通エリア骨格（テンプレート）

すべてのゾーンは、可能な限り **同じスロット構造** を持つ。

```
                    [N ゲート枠]  ※ 手前。枠に置けるのはゲートのみ（壁・川なし）
                         │
                    北カージナル道
                         │
     [W ゲート枠]── 中央広場 / ハブ ──[E ゲート枠]
                         │
                    南カージナル道
                         │
                    [S ゲート枠]  ※ 奥。壁・川は建物背面より外側

  外周（S/E/W）: 崖・壁・川（角は滝や岩などテーマ演出）
  外周（N / 手前）: 空。ゲートがあるときだけゲート（＋橋）
  内側四隅: アトラクション / Coming Soon / テーマデコ用スロット
```

### カージナル導線（L2・絶対確保）

| 軸 | 内容 | 禁止 |
|----|------|------|
| 東 | 中央 → 東橋 → 東ゲート | ベンチ長辺・鉢・建物本体 |
| 西 | 同上 | 同上 |
| 南 | 中央 → 南列アトラクション。奥外周は建物より外側 | 中央線を塞ぐデコ |
| 北 | 中央 → 北ゲート用中央線は空ける。**北外周はゲート以外なし** | 中央線を塞ぐデコ／北辺の川・壁・封印 |

幅の目安: 通路クリア **約 3+**（プレイヤー＋すれ違い）。ゲート前はさらに橋幅分。

**地面・家具の揃え**: 金縁（`pathEdgeX`）は側面アトラクションの **外側**。入口中心や建物本体を貫かない。ベンチ／鉢は壁際（縁の外〜川の手前）。  
**床の横幅は全区共通**（`SHARED_PARK_GROUND`: pathSize / paver / district）。差はテーマ色と `pathEdgeX` のみ。  
**橋**は川の真上だけ（園路に長く突き出さない）。門の ApproachPath は足元短尺。

### アトラクション・スロット（1 エリア最大 3）

四隅または南／北の「カージナルを切らない位置」に最大 3。  
**座標の正は §14**（Plaza A/B/C・Mountain 本棟＋ Coming Soon 2）。

| スロット | おおよその位置イメージ | 現行例 |
|----------|------------------------|--------|
| A | 南西寄り | Plaza: Find `(-16, -10.2)` |
| B | 南中央寄り（中央道は空けたまま） | Plaza: Trait `(0, -6.8)` / Mountain: Climb `(0, -7)` |
| C | 南東寄り | Plaza: Street `(16, -10.2)` |
| 予備 | Coming Soon 建設中棟 | Mountain: `(-12.5, -11.5)` / `(12.5, -11.0)` |

**満杯なら新エリア＋橋ゲート**。既存エリアを押し広げて 4 棟目をねじ込まない。

### ゲート枠（4 方位）

- 各辺に「ゲート枠」スロットを持つ（未使用なら壁／閉じた門／看板のみでも可）
- 実ゲートがある辺だけ橋を出し、柵／崖を開口
- 双方向: 行きの `targetSpawn` は、向こうエリアの **対向ゲート内側**（橋を降りた直後）
- **向き**: 到着時は地区の内側を向く（門・入口を背にする）
  - 東から到着 → `Math.PI / 2`（+X）／西から到着 → `-Math.PI / 2`（-X）
  - 南向きアトラクション復帰 → `0`（+Z）

## 3. エリア＝テーマ（差し替えレイヤー）

エリア追加時は **先にテーマを一文で決める**。

例:

| Zone | テーマ一文 | 地面・色 | 境界 | ゲート |
|------|------------|----------|------|--------|
| `plaza` | 夜のクラシック園 | 石畳・金・噴水 | 整った壁＋穏やかな水路 | 地区ごとの意匠門 |
| `mountain` | ボクセルの山岳地区 | 苔・土・岩・雪 | 岩崖＋急流／滝 | ボクセル山門 |

テーマが決まったら揃えるもの:

1. 地面色・園路色（ゾーン look / layout）
2. 外周の崖・川のマテリアル
3. L3 デコ（木・岩・ランプの種類）
4. ゲート theme メッシュ
5. Coming Soon / 看板トーン

**骨格座標は共有。変えるのはテーマスキン。**

## 4. ゾーン構成（現行）

Park は **ゾーン切替**（フルリロードしない）。描画・衝突はアクティブゾーンのみ。

| Zone ID | 名前 | 役割 | テーマ |
|---------|------|------|--------|
| `plaza` | Meebits Plaza | メインハブ | クラシック園 |
| `mountain` | Mountain District | 山岳／建設中 | ボクセルマウンテン |

### ハードルール

1. **1 ゾーンあたりプレイ可能ゲームは最大 3**
2. 満杯 → **新ゾーン + 橋 + ゲート**（押し込み禁止）
3. ゾーン間移動は **橋→ゲートのみ**
4. `?from=<attractionId>` 復帰は **所属ゾーンの入口前**（カージナルを塞がない位置）

- [ ] 接続元エリアの対応カージナルにゲート＋橋
- [ ] 双方向 `targetSpawn`

## 7. アトラクション追加

1. テーマとスロット（A/B/C）を決める — カージナルを切らない
2. ルート／`AttractionId` を別途
3. `TOP_ATTRACTIONS`（`zoneId`, footprint, 文言）
4. ランドマーク見た目（**ゾーンテーマに合わせる**）
5. `attractionIds` 更新
6. footprint ↔ 衝突
7. 入口前 L2 を確認
8. `?from=` 復帰

満杯なら手順 0 として **新区＋橋ゲート** を先に。

## 8. ゲート＆橋

| 項目 | 方針 |
|------|------|
| 体験 | 橋を渡る → ゲート意匠 → ゾーン切替 |
| 位置 | カージナル端のみ（斜め配置で軸を壊さない） |
| theme | 行き先エリアの世界観 |
| yaw | 基本は軸平行（真向き）。斜めは例外 |
| 衝突 | 柱・岩は円 or ローカル変換。巨大 AABB パッド禁止 |
| 開口 | 崖・壁・柵の開口と橋幅を一致 |

帰りゲートも同じ骨格の対向辺。マウンテン側など **真向きでよい** 場合は無理に斜めにしない。

## 9. 家具・デコ

壁際・四隅・カージナルの **脇** のみ。テーマ物件（岩、苔、ランプ種別）はゾーンの世界観に合わせる。  
**手前辺（N）の外周帯にはデコを置かない**（ゲート以外）。奥壁は建物背面と重ねない。

### 置き方（プロの揃え）— 現行デフォルトは §14

- **散らさない**。ベンチ＋季節オブジェは噴水／中央園路の **開けた帯** にだけ置く
- 街灯は **左右対称の均等グリッド**（ランダム禁止）
- **東西ゲート／封印門の周囲に家具を固めない**（門が主役）
- **建物と壁の隙間にベンチ・木をねじ込まない**
- 禁止帯の目安: `x>12`（東ゲート＋Street）/ `x<-16`（西封印門）/ `|x|<3.5`（中央 NS）/ 各入口前
- 木は手前〜中段の左右（中央導線の外）。衝突は `TREE_RADIUS`（`topCollisions.ts`）

## 10. 衝突

- ゾーン切替で `setParkCollisionZone`
- 見た目変更＝衝突更新
- 外周の崖・川は歩行不可帯として必ず登録（**手前クリア辺は衝突もスキップ**）
- 橋は歩ける帯、両脇の川は円／帯で落とさない（進入不可）
